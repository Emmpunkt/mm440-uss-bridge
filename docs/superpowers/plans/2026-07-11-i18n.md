### Task 1: config_store `language` + Migration + /api/config

**Files:** `include/config_store.h`, `src/config_store.cpp`, `src/net_web.cpp`

- [ ] **Step 1:** In `struct Config` **am Ende** ergänzen: `char language[3];`
- [ ] **Step 2:** `fillDefaults`: `copyStr(c.language, sizeof(c.language), "de");`
- [ ] **Step 3:** `configLoad()` migrationsfähig machen:
```cpp
void configLoad() {
  Preferences p; p.begin(NVS_NS, true);
  Config tmp; memset(&tmp, 0, sizeof(tmp));
  size_t got = p.getBytes(NVS_KEY, &tmp, sizeof(tmp));
  p.end();
  if (got > 0 && tmp.magic == CONFIG_MAGIC) {
    g_cfg = tmp;
    if (g_cfg.language[0] != 'd' && g_cfg.language[0] != 'e')  // fehlt (alter Blob)
      copyStr(g_cfg.language, sizeof(g_cfg.language), "de");
    g_cfg.version = CONFIG_VERSION;
  } else {
    fillDefaults(g_cfg);
  }
}
```
- [ ] **Step 4:** `net_web.cpp handleConfigGet`: `d["language"] = c.language;`
- [ ] **Step 5:** `net_web.cpp handleConfigPost`: nach den anderen Feldern:

```cpp
    copyStrCfg(c.language, sizeof(c.language), (strcmp(l,"en")==0) ? "en" : "de");
  }
```
(Helfer: es gibt bereits `setStr`-Lambda; nutze `setStr(c.language,...)` dann normalisieren, oder direkt `strncpy`. Sicherstellen: nur "de"/"en".)

### Task 3: Web-UI i18n (beide Seiten) + Settings-Dropdown

**Files:** `src/net_web.cpp` (INDEX_HTML, SETTINGS_HTML)

- [ ] **Step 1:** In `INDEX_HTML` und `SETTINGS_HTML` alle festen Labels mit `data-i18n="key"` versehen (Buttons, Überschriften, Feldbeschriftungen). Umschalter oben rechts: `<span id="lang" style="float:right;font-size:.8rem;cursor:pointer;color:#888">DE | EN</span>`.
- [ ] **Step 2:** JS-Wörterbuch + Anwendung (in beide Seiten):
```javascript
const I18N={de:{...},en:{...}};        // alle keys
function applyLang(l){document.documentElement.lang=l;localStorage.lang=l;
 document.querySelectorAll('[data-i18n]').forEach(e=>{const t=I18N[l][e.dataset.i18n];if(t)e.textContent=t;});
 window.LANG=l;}
```
Umschalter: klick toggelt de/en → `applyLang`. Startsprache: `localStorage.lang || (configLang) || 'de'` (configLang aus `/api/config`).
- [ ] **Step 3:** Web-Stör-/Warntext zweisprachig via JS-Codetabelle über `fault_num`/`warn_num`:
```javascript
const FAULTS={1:{de:'Überstrom',en:'Overcurrent'},...}, WARNS={922:{de:'Keine Last am Umrichter',en:'No load on inverter'},...};
function ftext(n){const e=FAULTS[n];return e?e[window.LANG]:(n?('F'+String(n).padStart(4,'0')):'');}
```
In `poll()` Stör-/Warnzeile darüber statt `s.fault_text`.
- [ ] **Step 4:** `SETTINGS_HTML`: Sprach-Dropdown `<select id="language"><option value="de">Deutsch</option><option value="en">English</option></select>` in einer eigenen Karte; in `F`-Liste + `load()`/`save()` aufnehmen.
- [ ] **Step 5:** Bauen, flashen. Verify: Hauptseite + `/settings` — Umschalter wechselt alle Labels sofort; Reload behält Sprache (localStorage); Stör-/Warntext folgt Sprache. Settings-Dropdown zeigt/speichert Gerätesprache.
