# Design: Sprachumschaltung DE/EN (UI + MQTT)

Datum: 2026-07-11
Projekt: MM440 USS Bridge (ESP32-C3)

## Ziel

Das gesamte bedienbare System zweisprachig (Deutsch/Englisch) machen:
- **Web-UI**: Sofort-Umschalter „DE | EN" (pro Browser, `localStorage`).
- **MQTT/HA**: Entitätsnamen + Stör-/Warntexte in der **Gerätesprache**.
- Eine **Gerätesprache** auf `/settings` setzt MQTT/HA und den Web-Standard.

Speicher ist unkritisch: ~231 KB Flash frei; die zweite Sprache kostet < 10 KB.

## Entscheidungen (mit Nutzer abgestimmt)

- Umfang: UI-Labels **und** Stör-/Warn-Klartexte.
- MQTT/HA ebenfalls Englisch möglich → Sprache muss serverseitig wählbar sein
  (ein HA-Entitätsname ist einsprachig).
- Web: Sofort-Umschalter (pro Browser) **plus** Gerätesprache (Default).
- Default-Sprache: **de**.

## Architektur

### Zwei Sprachquellen

- **Firmware-Strings** (für MQTT): Entitätsnamen + Fehler-/Warntabelle
  zweisprachig; Auswahl über `cfg.language`.
- **JS-Wörterbuch** (fürs Web): dieselben Labels + Code→Text-Tabelle, damit der
  Browser ohne Server-Roundtrip sofort umschaltet.

### `config_store` — neues Feld + schonende Migration

`struct Config` bekommt **am Ende** `char language[3]` ("de"/"en").

Migration (kein Datenverlust): `configLoad()` liest den NVS-Blob in ein zuvor
genulltes `tmp`. Akzeptiert wird jeder Blob mit passendem `magic` (auch der
*alte*, kürzere ohne `language`). Fehlt `language` (Byte 0 == 0), wird "de"
gesetzt; `version` wird auf aktuell gehoben. So bleiben MQTT-Host, Gerätename
usw. erhalten. `configSave()` schreibt weiterhin die volle Struct.

### `mm440_faults` — zweisprachig

```
struct CodeText { uint16_t code; const char* de; const char* en; };
const char* faultText(uint16_t code, bool en);
const char* warnText(uint16_t code, bool en);
String faultLabel(uint16_t code, bool en);   // Text oder "F%04u"
String warnLabel(uint16_t code, bool en);
```
Kuratierte Tabelle bekommt je Code einen EN-Text.

### `net_mqtt` — Namen/Texte nach `cfg.language`

- `mqttBegin` merkt sich `langEn = (cfg.language == "en")`.
- `publishDiscovery`: jeder Entitätsname wählt de/en (kleine Helfer-Tabelle bzw.
  Ternäre je Entität). Englische Namen: Actual frequency, Fault, Warning, Mains
  contactor, Motor, Setpoint frequency, Acknowledge fault, Motor current,
  DC-link voltage, Output voltage, Fault text, Warning text.
- `publishState`: `fault_text`/`warn_text` über `faultLabel/warnLabel(code, langEn)`.
- Sprachwechsel wird durch Speichern→Reboot wirksam (Discovery neu publiziert).

### `net_web`

- `/api/config` GET liefert `language`; POST akzeptiert es (Validierung: nur
  "de"/"en", sonst "de").
- `/settings`: Dropdown **Sprache** (Deutsch/English).
- `INDEX_HTML` + `SETTINGS_HTML`: alle festen Labels tragen `data-i18n="key"`.
  Ein JS-Objekt `I18N = { de:{…}, en:{…} }` hält die Strings. `applyLang(l)`
  setzt alle `[data-i18n]`-Texte. Umschalter „DE | EN" oben rechts.
  Startsprache: `localStorage.lang` falls gesetzt, sonst `configLanguage` aus
  `/api/config`, sonst "de".
- Web-Stör-/Warntext: JS-Tabelle (de/en) über `fault_num`/`warn_num` aus
  `/api/status` (die Firmware-`fault_text` wird vom Web ignoriert; MQTT nutzt sie
  weiter).

## Datenfluss

- Web-Sprachwahl: Toggle → `applyLang` (sofort) + `localStorage`. Unabhängig vom
  Gerät, pro Browser.
- Gerätesprache: `/settings` → `POST /api/config {language}` → NVS → Reboot →
  MQTT-Discovery + Web-Default in neuer Sprache.

## Fehlerbehandlung

- Ungültiger `language`-Wert → "de".
- Alter NVS-Blob ohne `language` → migriert, "de".
- Unbekannter Fehler-/Warncode → Fallback „F0123"/„A0501" (sprachneutral).

## Nicht im Scope (YAGNI)

- Weitere Sprachen (Struktur erlaubt Erweiterung).
- Übersetzung frei konfigurierbarer Werte (Gerätename bleibt wie eingegeben).
- Serielle Debug-Ausgaben (bleiben knapp/englischnah, kein UI).

## Test (Hardware + HA-MCP)

1. Build/Flash. **Bestehende Config bleibt erhalten** (MQTT-Host noch gesetzt,
   `/api/config` zeigt alte Werte + `language:"de"`).
2. Web: Umschalter DE↔EN wechselt alle Labels sofort; Auswahl überlebt Reload
   (localStorage). Stör-/Warntext folgt der Sprache (per `warn_num`).
3. Gerätesprache auf „en" via `/settings` → Reboot → `ha_search` zeigt Entitäten
   mit **englischen Namen**; 5-Hz-Lauf → Warnungstext „No load on inverter".
4. Zurück auf „de" → Namen wieder deutsch.

## Branch

`feature/i18n` (auf `master`).
